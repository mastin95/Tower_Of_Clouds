---
name: Always persist ideas to file immediately
description: Every design idea discussed in conversation MUST be written to a design file before moving on — never leave decisions only in conversation
type: feedback
---

Every idea, decision, or design concept discussed in conversation MUST be written to the appropriate file in `design/` before closing the topic or moving on. The conversation is ephemeral — only files persist across sessions and machines.

**Why:** The user lost an entire discussion about an ability unlock system because it was only in conversation and they moved to a different PC. The session state and conversation don't transfer between machines.

**How to apply:** After discussing any design idea, immediately ask "May I write this to [filepath]?" and persist it. Don't wait for the full idea to be "finished" — write drafts, write sketches, write bullet points. Something on disk is infinitely better than a perfect idea in a lost conversation.
